準備程式設計競賽（Competitive Programming, CP）是一場結合**演算法邏輯**、**資料結構熟悉度**與**程式實作速度**的馬拉松。

根據你目前的學習背景與目標（如 CPE、ZeroJudge、或是未來的 ICPC/CCPC），以下是一份系統性的準備指南：

---

## 1. 穩固語言與基本功

在競賽中，**C++** 是最主流的語言，因為其 STL（Standard Template Library）功能強大且執行效率高。

- **精通 C++ STL：** 必須熟練使用 `vector`, `set`, `map`, `priority_queue`, `sort`, `lower_bound` 等工具。

- **時間與空間複雜度：** 學會估算 $O(n)$, $O(n \log n)$, $O(2^n)$ 等複雜度。競賽中通常 1 秒能跑 $10^8$ 次運算，這決定了你該選哪種演算法。

- **實作細節：** 練習處理邊界條件（Boundary conditions）、大數運算、以及精確的輸入輸出處理（如 `scanf/printf` 與 `cin/cout` 的速度差異）。

---

## 2. 核心演算法分類突破

不要盲目刷題，建議按主題進攻：

| **類別**   | **重點演算法 / 資料結構**                                      |
| -------- | ----------------------------------------------------- |
| **基礎技巧** | 遞迴（Recursion）、二分搜尋（Binary Search）、貪婪演算法（Greedy）       |
| **動態規劃** | 背包問題（Knapsack）、最長共同子序列（LCS）、LIS、區間 DP                 |
| **圖論**   | BFS/DFS、最短路徑（Dijkstra, Floyd-Warshall）、最小生成樹（Kruskal） |
| **數學**   | 質數篩法、大數運算、組合數學（C(n, k)）、**鴿籠原理**、排容原理                 |
| **資料結構** | 樹狀數組（BIT）、線段樹（Segment Tree）、並查集（Union-Find）           |

---

## 3. 刷題平台與資源推薦

針對你目前的環境，建議循序漸進：

- **新手入門（台灣常用）：**
  
  - **ZeroJudge：** 適合練習基礎題目與 CPE 考古題。
  
  - **CPE 官網：** 針對「大學程式能力檢定」，務必熟練其「精選 49 題」。

- **進階訓練（國際主流）：**
  
  - **CSES Problem Set：** 非常推薦！這套題目涵蓋了 CP 最核心的基礎知識，題目精簡且具代表性。
  
  - **Codeforces：** 定期舉辦比賽。練習在壓力下限時解題，並提升對題目的敏感度（Rating）。
  
  - **AtCoder：** 題目品質極高，適合練習思維邏輯。

- **經典教材：**
  
  - *Introduction to Algorithms (CLRS)*：演算法聖經。
  
  - *Competitive Programming 4* (Steven Halim)：專為競賽撰寫的百科全書。

---

## 4. 實戰訓練策略

- **模擬比賽：** 每週固定抽出 3-5 小時，關掉通訊軟體，模擬真實比賽限時解題。

- **補題（Upsolving）：** 比賽沒寫出來的題目，**賽後一定要看懂解答並親手寫過**。這才是進步最快的時刻。

- **學習 Debug：** 練習手寫測資（Corner cases），學會利用 `assert()` 或觀察變數值快速定位錯誤。

- **整理模板（Codebook）：** 準備一份自己的演算法模板，包含常用的複雜資料結構（如線段樹、網路流），在允許攜帶參考資料的比賽中非常實用。

---

## 5. 心態建設

程式競賽本質上是「腦力體操」。遇到卡關（WA/TLE）是常態，保持對邏輯推演的熱情，從報錯訊息中學習，比起拿到 AC（Accepted）更有價值。
